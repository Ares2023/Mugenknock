【公式試験ガイド概要 (AIF-C01)】
■ 試験の配分
- 第1ドメイン：AIとMLの基礎 (20%)
- 第2ドメイン：生成AIの基礎 (24%)
- 第3ドメイン：基盤モデルのアプリケーション (28%)
- 第4ドメイン：責任あるAIのガイドライン (14%)
- 第5ドメイン：AIソリューションのセキュリティ、コンプライアンス、ガバナンス (14%)

■ 公式タスク文（出題で必ずカバーすること）
【第1ドメイン：AIとMLの基礎】
- Task 1.1: 基本的なAIの概念と用語を説明する（教師あり/なし学習・強化学習・ディープラーニング・ニューラルネットワーク・推論・精度・適合率・再現率）
- Task 1.2: AIの実際のユースケースを特定する（画像認識・自然言語処理・予測・レコメンデーション・異常検知・需要予測）
- Task 1.3: ML開発ライフサイクルを説明する（データ収集・前処理・特徴エンジニアリング・モデルトレーニング・評価・デプロイ・モニタリング）

【第2ドメイン：生成AIの基礎】
- Task 2.1: 生成AIの基本概念を説明する（大規模言語モデル（LLM）・基盤モデル・Transformer・トークン・埋め込み・ハルシネーション・コンテキストウィンドウ）
- Task 2.2: ビジネス課題解決における生成AIの能力と限界を理解する（テキスト生成・コード生成・画像生成・要約・翻訳・ハルシネーションリスク・データの鮮度）
- Task 2.3: 生成AIアプリケーション構築のためのAWSインフラストラクチャと技術を説明する（Amazon Bedrock・基盤モデル選択・API経由の推論・モデルカスタマイズ）

【第3ドメイン：基盤モデルのアプリケーション】
- Task 3.1: 基盤モデルベースのアプリケーションの設計上の考慮事項を説明する（RAG・エージェント・チェーン・Knowledge Bases・Guardrails・コスト最適化）
- Task 3.2: 効果的なプロンプトエンジニアリング技術を選択する（ゼロショット・フューショット・チェーンオブソート・ロールプロンプティング・システムプロンプト）
- Task 3.3: 基盤モデルのトレーニングとファインチューニングプロセスを説明する（ファインチューニング・RLHF・PEFT・LoRA・知識蒸留・継続事前学習）
- Task 3.4: 基盤モデルのパフォーマンスを評価する方法を説明する（ROUGE・BLEU・BERTScore・ヒューマン評価・ベンチマーク・A/Bテスト）

【第4ドメイン：責任あるAIのガイドライン】
- Task 4.1: 責任あるAIシステムの開発を説明する（公平性・透明性・説明可能性・プライバシー・セキュリティ・AIバイアス・データ品質・モデルカード）
- Task 4.2: 透明性と説明可能なモデルの重要性を認識する（XAI・SHAP・LIME・特徴量重要度・モデル解釈可能性・規制コンプライアンス）

【第5ドメイン：AIソリューションのセキュリティ、コンプライアンス、ガバナンス】
- Task 5.1: AIシステムをセキュリティ保護する方法を説明する（IAM・プロンプトインジェクション対策・Guardrails・モデルアクセス制御・データ暗号化・VPC）
- Task 5.2: AIシステムのガバナンスとコンプライアンス規制を認識する（GDPR・HIPAA・EU AI Act・AWS AI Service Cards・Model Risk Management）

■ 出題範囲の主要サービス
- 生成AI: Amazon Bedrock（基盤モデル選択・Knowledge Bases・Agents・Guardrails・Model Evaluation）, PartyRock
- ML開発: SageMaker（Studio・Autopilot・Ground Truth・Model Monitor・Pipeline・JumpStart）
- AI専用サービス: Rekognition, Comprehend, Textract, Polly, Transcribe, Lex, Translate, Forecast, Personalize, Kendra, Q Business
- インフラ: EC2 (GPU/Inferentia インスタンス), Lambda, ECS, SageMaker Inference
