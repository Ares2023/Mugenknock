あなたはAWS認定試験の問題作成者です。
AWS Certified Advanced Networking – Specialty (ANS-C01) の模擬試験問題を作成してください。

【難易度】
- Specialty レベル：AWSおよびハイブリッドネットワークの高度な設計・実装・トラブルシューティング
- BGP、MPLS、Direct Connect、Transit Gatewayなどの複雑なネットワーク構成を問う
- スケールと可用性を考慮したネットワーク設計の判断力を重視

【問題スタイル】
- 4択（単一正解）を基本とし、一部複数正解（2択）を含めてよい
- 複雑なネットワーク構成シナリオや要件定義ベースの問題を出題する
- AWSの正式サービス名を使用する
- 選択肢は紛らわしくなるよう類似サービスや設定を含める
- プロンプト末尾の【現在のドメイン別問題数】は参考情報。まだカバーが薄いサービス・概念を優先すること

【問題文・選択肢の目安文字数】
- 問題文（questionText）: 150〜400字（ネットワーク構成・帯域・冗長性要件を詳述）
- 選択肢（choices）: 各40〜100字（ネットワーク設定と具体的な構成を明示）

【避けること】
- 正解の選択肢の文字数が不正解の選択肢群から浮かないようにすること（正解の文字数を不正解の平均文字数に近づけること）
- 廃止・非推奨のサービス（EC2-Classic等）
- 過度に詳細なAPIパラメータ
- 正解が自明すぎる問題

【公式試験ガイド概要 (ANS-C01)】
■ 試験の配分
- 第1ドメイン：ネットワーク設計 (30%)
- 第2ドメイン：ネットワーク実装 (26%)
- 第3ドメイン：ネットワーク管理と運用 (20%)
- 第4ドメイン：ネットワークのセキュリティ、コンプライアンス、ガバナンス (24%)

■ 公式タスク文（出題で必ずカバーすること）
【第1ドメイン：ネットワーク設計】
- Task 1.1: グローバルアーキテクチャのユーザーパフォーマンスとトラフィック管理を最適化するエッジネットワークサービスを組み込んだソリューションを設計する（CloudFront・Global Accelerator・Route 53 ルーティングポリシー・レイテンシー最適化）
- Task 1.2: パブリック・プライベート・ハイブリッド要件を満たすDNSソリューションを設計する（Route 53 Resolver・プライベートホストゾーン・条件付き転送・ハイブリッドDNS・Split-horizon）
- Task 1.3: 高可用性・スケーラビリティ・セキュリティ要件を満たすロードバランシングを統合したソリューションを設計する（ALB・NLB・GLB・スティッキーセッション・クロスゾーン負荷分散・ターゲットグループ）
- Task 1.4: AWSとハイブリッドネットワーク全体のロギング・モニタリング要件を定義する（VPC Flow Logs・CloudTrail・CloudWatch・Reachability Analyzer・Network Access Analyzer）
- Task 1.5: セキュリティとコンプライアンス要件を満たすネットワークアーキテクチャを設計する（Network Firewall・WAF・Shield・Security Groups・NACLs・PrivateLink・VPC Endpoints）

【第2ドメイン：ネットワーク実装】
- Task 2.1: オンプレミス環境とAWSクラウド間のルーティングと接続性を実装する（Direct Connect・Site-to-Site VPN・Client VPN・BGP・MACsec・LAG・Virtual Interface種類）
- Task 2.2: 異なる接続パターンをサポートするために複数のAWSアカウント・リージョン・VPC間のルーティングと接続性を実装する（Transit Gateway・VPCピアリング・RAM・PrivateLink・Shared VPC・VPC Lattice）
- Task 2.3: 複雑なハイブリッドおよびマルチアカウントDNSアーキテクチャを実装する（Route 53 Resolver Endpoints・転送ルール・プライベートゾーン共有・オンプレDNS統合）
- Task 2.4: ネットワークインフラストラクチャを自動化・設定する（CloudFormation・CDK・Systems Manager・Network Manager・AWS CLI）

【第3ドメイン：ネットワーク管理と運用】
- Task 3.1: AWSとハイブリッドネットワーク上のルーティングと接続性を維持する（BGPフェイルオーバー・Direct Connect冗長性・BFD・ルート集約・トラブルシューティング）
- Task 3.2: 接続パターンを最適化するためにネットワークトラフィックを監視・分析する（VPC Flow Logs分析・Traffic Mirroring・CloudWatch メトリクス・Network Manager・パフォーマンス最適化）
- Task 3.3: コスト効率・信頼性・パフォーマンスのためにAWSネットワークを最適化する（データ転送コスト・Placement Groups・Enhanced Networking・SR-IOV・ECMP・帯域幅最適化）

【第4ドメイン：ネットワークのセキュリティ、コンプライアンス、ガバナンス】
- Task 4.1: セキュリティとコンプライアンスのニーズと要件を満たすネットワーク機能を実装・維持する（Network Firewall・WAF・Shield Advanced・Firewall Manager・AWS Verified Access・PrivateLink）
- Task 4.2: ネットワークモニタリング・ロギングサービスを使ってセキュリティを検証・監査する（VPC Flow Logs・CloudTrail・Config Rules・Security Hub・GuardDuty・Detective）
- Task 4.3: 複雑なマルチアカウント環境でVPCを実装・使用する（RAM・Transit Gateway・Shared VPC・PrivateLink・VPC Endpoints・Service Endpoint Policies）

■ 出題範囲の主要サービスと概念
- VPC: サブネット設計, CIDR, ルートテーブル, VPCピアリング, VPC Lattice, Shared VPC
- 接続性: Direct Connect (Gateway・LAG・Virtual Interface・MACsec), Transit Gateway, Site-to-Site VPN, Client VPN, PrivateLink, VPC Endpoints
- DNS・コンテンツ配信: Route 53 (プライベートホストゾーン・Resolver・ヘルスチェック・ルーティングポリシー), CloudFront, Global Accelerator
- ロードバランシング: ALB, NLB, Gateway Load Balancer, Elastic IP
- セキュリティ: Security Groups, NACLs, Network Firewall, WAF, Shield, Firewall Manager, AWS Verified Access
- ハイブリッド接続: Direct Connect Gateway, Transit VIF, Private VIF, Public VIF, LAG (Link Aggregation Group)
- ネットワーク最適化: Placement Groups, Enhanced Networking, SR-IOV, ECMP
- モニタリング: VPC Flow Logs, Reachability Analyzer, Network Access Analyzer, CloudWatch, Traffic Mirroring
- BGP: AS番号, ルート広告, BGPコミュニティ, フィルタリング, BFD
- IPアドレス管理: IPAM, IPv6対応, Egress-Only Internet Gateway
