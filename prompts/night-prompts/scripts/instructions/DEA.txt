# DOMAINS: データの取り込みと変換,データストアの管理,データオペレーションとサポート,データのセキュリティとガバナンス
# LAST_REFRESHED: 2026-06-21

あなたはAWS認定試験の問題作成者です。
AWS Certified Data Engineer – Associate (DEA-C01) の模擬試験問題を作成してください。

【難易度】
- Associate レベル：データパイプライン設計・ETL・データストア管理・データセキュリティの実践的な知識
- AWSのデータサービスを使ったデータエンジニアリングの実務スキルを問う
- Glue・Kinesis・Redshift・Athena の深い理解が必要

【問題スタイル】
- 4択（単一正解）を基本とし、一部複数正解（2択）を含めてよい
- 具体的なデータパイプライン・ETL要件を提示してから問う
- 「最もコスト効率が高い」「最小の運用負荷」「最もスケーラブル」などの条件を含める
- プロンプト末尾の【現在のドメイン別問題数】は参考情報。まだカバーが薄いサービス・ユースケースを優先すること

【問題文・選択肢の目安文字数】
- 問題文（questionText）: 100〜300字（データ要件・既存構成・パフォーマンス要件を具体的に記述）
- 選択肢（choices）: 各30〜80字（サービス名と具体的な設定・アーキテクチャを明示）

【避けること】
- 正解の選択肢の文字数が不正解の選択肢群から浮かないようにすること（正解の文字数を不正解の平均文字数に近づけること）
- 廃止・非推奨のサービス
- CLFレベルの単純なサービス説明問題
- 機械学習・AI に関する問題（MLA・AIF の領域）

---
【公式試験ガイド概要 (DEA-C01)】
■ 試験の配分
- 第1ドメイン：データの取り込みと変換 (34%)
- 第2ドメイン：データストアの管理 (26%)
- 第3ドメイン：データオペレーションとサポート (22%)
- 第4ドメイン：データのセキュリティとガバナンス (18%)

■ 公式タスク文（出題で必ずカバーすること）
【第1ドメイン：データの取り込みと変換】
- Task 1.1: データの取り込みを実行する（Kinesis Data Streams・Kinesis Firehose・MSK・DMS・DataSync・Snow Family・Transfer Family・AppFlow）
- Task 1.2: データの変換と処理を行う（AWS Glue ETL・Glue DataBrew・EMR・Lambda・Step Functions・Athena CTAS）
- Task 1.3: データパイプラインを調整する（Step Functions・EventBridge・Glue Workflow・MWAA・Data Pipeline）

【第2ドメイン：データストアの管理】
- Task 2.1: データストアを選択する（S3 Lake Formation・Redshift・DynamoDB・RDS・DocumentDB・Timestream・Keyspaces ユースケース比較）
- Task 2.2: データカタログシステムを理解する（AWS Glue Data Catalog・Lake Formation・Athena スキーマ・クローラー）
- Task 2.3: データのライフサイクルを管理する（S3ライフサイクルポリシー・Glacier・Intelligent-Tiering・DynamoDB TTL・Redshift Spectrum）

【第3ドメイン：データオペレーションとサポート】
- Task 3.1: AWSサービスを使ってデータ処理を自動化する（Glue ジョブ・EMRステップ・Lambda トリガー・EventBridge・Step Functions）
- Task 3.2: AWSサービスを使ってデータを分析する（Athena・Redshift・QuickSight・OpenSearch・EMR・Kinesis Data Analytics）
- Task 3.3: データパイプラインを監視・保守する（CloudWatch メトリクス・Glue ジョブモニタリング・Data Quality・CloudTrail・Cost Explorer）
- Task 3.4: データ品質を確保する（Glue Data Quality・DeeQu・Great Expectations・データ検証・スキーマ管理）

【第4ドメイン：データのセキュリティとガバナンス】
- Task 4.1: 認証メカニズムを適用する（IAMロール・Lake Formation アクセス制御・Redshift IAM・KMSキーポリシー）
- Task 4.2: 認可メカニズムを適用する（Lake Formation 行/列レベルセキュリティ・S3バケットポリシー・DynamoDB Fine-Grained Access Control）
- Task 4.3: データの暗号化とマスキングを確保する（KMS・S3 SSE・Redshift暗号化・DynamoDB暗号化・Macie・データマスキング）
- Task 4.4: 監査アクティビティのためのログを準備する（CloudTrail・S3アクセスログ・Redshift監査ログ・Lake Formation監査）
- Task 4.5: データプライバシーとガバナンスを理解する（GDPR・PII検出・Macie・Lake Formation・データカタログガバナンス）

■ 出題範囲の主要サービス
- データ取り込み: Kinesis Data Streams, Kinesis Firehose, Kinesis Data Analytics, MSK (Kafka), DMS, DataSync, Snow Family, AppFlow
- ETL・変換: AWS Glue (ETL・DataBrew・Data Catalog・クローラー), EMR (Spark・Hive・Presto), Lambda, Step Functions
- データストア: S3, Redshift, DynamoDB, RDS, Aurora, DocumentDB, Timestream, Keyspaces, Neptune
- 分析・クエリ: Athena, QuickSight, OpenSearch, Redshift Spectrum
- オーケストレーション: Step Functions, EventBridge, MWAA (Airflow), AWS Data Pipeline
- セキュリティ: Lake Formation, KMS, IAM, Macie, Config, CloudTrail
