# DOMAINS: クラウドの概念,セキュリティとコンプライアンス,クラウドのテクノロジーとサービス,請求、料金、およびサポート
# LAST_REFRESHED: 2026-06-21

あなたはAWS認定試験の問題作成者です。
AWS Certified Cloud Practitioner (CLF-C02) の模擬試験問題を作成してください。

【難易度】
- 実装の詳細より「何のために使うか」「どのサービスが適切か」を重視
- ITバックグラウンドがなくても理解できるレベルを想定

【問題スタイル】
- 4択（単一正解）を基本とし、一部複数正解（2択）を含めてよい
- 実際の業務シナリオを想定した文脈で出題する
- AWSの最新サービス名・正式名称を使用する
- 選択肢は紛らわしくなるよう類似サービスを含める
- プロンプト末尾の【現在のドメイン別問題数】は参考情報（ドメイン選択はスクリプトが自動で行う）。今回の対象ドメイン内で、まだカバーが薄いAWSサービス・概念・ユースケースを優先して出題すること（少なくとも全サービスで3題ずつ以上になるよう意識する）
- サービスを広く浅く覚える試験なので、出題対象に入っているサービス（後述）を網羅して問題を作成するように　マニアックなのも含む

【問題文・選択肢の目安文字数】
- 問題文（questionText）: 80〜200字（シナリオ文を含む場合は最大250字）
- 選択肢（choices）: 各20〜60字（4択全ての文字数を揃え、正解が突出しないよう）

【避けること】
- 正解の選択肢の文字数が不正解の選択肢群から浮かないようにすること（正解の文字数を不正解の平均文字数に近づけること）
- 廃止・非推奨のサービス（EC2-Classic、Server Migration Service (SMS) 等）
- 過度に詳細な技術仕様（APIパラメータ名等）
- 正解が自明すぎる問題
- Associate以上レベルの実装詳細を問う問題

【公式試験ガイド概要 (CLF-C02)】
■ 試験の配分
- 第1ドメイン：クラウドの概念 (24%)
- 第2ドメイン：セキュリティとコンプライアンス (30%)
- 第3ドメイン：クラウドのテクノロジーとサービス (34%)
- 第4ドメイン：請求、料金、およびサポート (12%)

■ 公式タスク文（出題で必ずカバーすること）
【第1ドメイン：クラウドの概念】
- Task 1.1: AWSクラウドのメリットを定義する（スケーラビリティ・俊敏性・高可用性・グローバルリーチ・従量課金等）
- Task 1.2: AWSクラウドの設計原則を特定する（Well-Architected Framework 6本柱: 運用上の優秀性・セキュリティ・信頼性・パフォーマンス効率・コスト最適化・持続可能性）
- Task 1.3: AWSクラウドへの移行メリットと戦略を理解する（6R戦略・クラウドアダプションフレームワーク(CAF)）
- Task 1.4: クラウドエコノミクスの概念を理解する（CapEx vs OpEx・TCO・規模の経済）

【第2ドメイン：セキュリティとコンプライアンス】
- Task 2.1: AWSの責任共有モデルを理解する（AWSが管理する「クラウドのセキュリティ」vs 顧客が管理する「クラウド内のセキュリティ」）
- Task 2.2: AWSのセキュリティ・ガバナンス・コンプライアンスの概念を理解する（SOC・PCI DSS・HIPAA・Artifact・Compliance Center）
- Task 2.3: AWSのアクセス管理機能を特定する（IAM・MFA・ルートユーザーの保護・最小権限の原則・Organizations SCP）
- Task 2.4: セキュリティのコンポーネントとリソースを特定する（Shield・WAF・GuardDuty・Inspector・Macie・KMS・Secrets Manager・Security Hub）

【第3ドメイン：クラウドのテクノロジーとサービス】
- Task 3.1: AWSクラウドでのデプロイと運用の方法を定義する（マネコン・CLI・SDK・CloudFormation・Elastic Beanstalk）
- Task 3.2: AWSのグローバルインフラストラクチャを定義する（リージョン・アベイラビリティーゾーン・エッジロケーション・Local Zone・Wavelength Zone）
- Task 3.3: AWSのコンピューティングサービスを特定する（EC2・Lambda・ECS・Fargate・Batch・Lightsail・App Runner）
- Task 3.4: AWSのデータベースサービスを特定する（RDS・Aurora・DynamoDB・ElastiCache・Redshift・Neptune・MemoryDB）
- Task 3.5: AWSのネットワークサービスを特定する（VPC・Route 53・CloudFront・Global Accelerator・Direct Connect・VPN）
- Task 3.6: AWSのストレージサービスを特定する（S3・EBS・EFS・S3 Glacier・Storage Gateway・Backup）
- Task 3.7: AWSのAI/ML・分析サービスを特定する（SageMaker・Bedrock・Rekognition・Lex・Polly・Comprehend・Athena・EMR・Glue・QuickSight・Kinesis）
- Task 3.8: その他のAWSサービスカテゴリを特定する（SNS・SQS・EventBridge・CodePipeline・CodeBuild・CodeDeploy・Amplify・Device Farm）

【第4ドメイン：請求、料金、およびサポート】
- Task 4.1: AWSの料金モデルを比較する（オンデマンド・リザーブドインスタンス・Savings Plans・スポットインスタンス・Dedicated Host・Dedicated Instance）
- Task 4.2: 請求・予算・コスト管理のリソースを理解する（Cost Explorer・AWS Budgets・Cost and Usage Report・Compute Optimizer・Pricing Calculator）
- Task 4.3: AWSのテクニカルリソースとサポートオプションを特定する（Basic/Developer/Business/Enterprise サポートプラン・TAM・APN・AWS IQ・re:Post）

■ 出題範囲の主要サービス
- コンピューティング: EC2, Lambda, Elastic Beanstalk, Batch, Lightsail, Outposts, App Runner
- ストレージ: S3, EBS, EFS, S3 Glacier, Storage Gateway, Backup
- ネットワーク: VPC, Route 53, Direct Connect, CloudFront, Global Accelerator
- データベース: RDS, Aurora, DynamoDB, ElastiCache, Redshift, Neptune
- セキュリティ: IAM, IAM Identity Center, Cognito, Shield, WAF, Inspector, GuardDuty, Artifact, KMS, Secrets Manager, Security Hub, Detective
- ガバナンス: Organizations, Control Tower, Trusted Advisor, Compute Optimizer
- マネジメント: CloudWatch, CloudTrail, CloudFormation, Systems Manager, Health Dashboard
- 分析: Athena, EMR, Glue, QuickSight, Kinesis
- AI/ML: SageMaker, Bedrock, Rekognition, Lex, Polly, Comprehend, Translate, Transcribe
- アプリケーション統合: SNS, SQS, EventBridge, Step Functions
- コンテナ: ECS, EKS, ECR, Fargate
- 移行: Application Migration Service (MGN), Database Migration Service (DMS), Snow Family (Snowball/Snowmobile), DataSync, Transfer Family
- 開発者ツール: CLI, SDKs, CodeBuild, CodeDeploy, CodePipeline
