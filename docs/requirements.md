# 要件定義書
# AWS資格問題サービス　要件定義書

---

## 1. サービス概要

| 項目 | 内容 |
|---|---|
| サービス内容 | AWS資格（CLF / SAA / SAP）の練習問題サービス |
| 対象言語 | 日本語のみ |
| 問題形式 | 単一選択・複数選択 |
| 問題管理者 | 管理者のみ（管理画面なし、DynamoDB直接操作） |

---

## 2. インフラ・技術スタック

| レイヤー | 技術 |
|---|---|
| ホスティング | AWS Amplify |
| ソース管理 | AWS CodeCommit |
| 認証 | Amazon Cognito（メール＋パスワード） |
| データベース | Amazon DynamoDB |
| API | Amazon API Gateway + AWS Lambda |
| メール送信 | Amazon SES |

---

## 3. 機能要件

### 3-1. 認証

- メールアドレス＋パスワードによるアカウント作成・ログイン
- パスワードリセット機能
- ログイン済みユーザーのみ：回答記録・統計閲覧・通報機能
- 未ログインユーザーでも：問題一覧の閲覧・コピー・CSVエクスポートが可能

---

### 3-2. 出題モード

#### 演習モード

| 項目 | 内容 |
|---|---|
| 問題数 | ユーザーが任意に指定 |
| 時間制限 | なし |
| 正誤表示 | 1問解答するごとに即時表示 |
| 解説表示 | 正誤表示と同時に表示 |
| 合否判定 | あり（正答率65%以上で合格） |

#### 模試モード

| 資格 | 問題数 | 制限時間 |
|---|---|---|
| CLF | 65問 | 90分 |
| SAA | 65問 | 130分 |
| SAP | 75問 | 180分 |

| 項目 | 内容 |
|---|---|
| 問題数・時間 | 変更不可（資格ごとに固定） |
| 分野比率 | 各資格の公式出題比率に準拠（下記参照） |
| 正誤表示 | 全問終了後にまとめて表示 |
| 合否判定 | あり（正答率65%以上で合格） |
| 途中再開 | 対応（ログイン時に未完了セッションを検出し再開可能） |

#### 模試モード　公式出題比率

**CLF**
| 分野 | 比率 |
|---|---|
| クラウドのコンセプト | 24% |
| セキュリティとコンプライアンス | 30% |
| クラウドテクノロジーとサービス | 34% |
| 請求・料金・サポート | 12% |

**SAA**
| 分野 | 比率 |
|---|---|
| セキュアなアーキテクチャの設計 | 30% |
| 弾力性に優れたアーキテクチャの設計 | 26% |
| 高パフォーマンスなアーキテクチャの設計 | 24% |
| コスト最適化されたアーキテクチャの設計 | 20% |

**SAP**
| 分野 | 比率 |
|---|---|
| 組織の複雑さに対応したソリューションの設計 | 26% |
| 新しいソリューションの設計 | 29% |
| 既存ソリューションの継続的改善 | 25% |
| ワークロードの移行とモダナイゼーション | 20% |

---

### 3-3. 出題フィルタ

| フィルタ | 内容 |
|---|---|
| 分野フィルタ | タグ（分野）を指定して出題対象を絞り込む |
| 不正解数フィルタ | 過去の不正解回数を基準に苦手問題を絞り込む |
| ランダム出題 | 問題の順序をシャッフルして出題 |

---

### 3-4. 学習履歴・統計

- 問題ごとの正誤を全履歴として蓄積（何回正解・何回不正解）
- タグ（分野）ごとの正答率を集計・表示
- 苦手分野ランキングの表示

---

### 3-5. コピー・エクスポート機能

| 機能 | 内容 |
|---|---|
| コピーボタン | 問題ごとにワンクリックでクリップボードにコピー |
| CSV出力 | チェックボックスで問題を選択しCSV出力（全選択あり） |
| CSVの内容 | 問題文・解答・解説 |

---

### 3-6. 通報機能

- 問題単位で通報ボタンを設置
- 通報内容を管理者の固定メールアドレスにSESで送信

---

## 4. データモデル設計（DynamoDB）

### アクセスパターン

| # | 操作 |
|---|---|
| 1 | 試験種別（CLF/SAA/SAP）で問題を絞り込む |
| 2 | 分野タグで問題を絞り込む |
| 3 | ユーザーの不正解数で問題を絞り込む |
| 4 | 問題を解いたとき正誤を記録する |
| 5 | ユーザーの問題ごとの正解・不正解回数を取得する |
| 6 | ユーザーの分野ごとの正答率を取得する |
| 7 | 模試セッションの開始・終了・結果を記録する |
| 8 | 問題を通報する |

---

### テーブル構成

#### ① Questions

| 属性 | 説明 |
|---|---|
| PK: `questionId` | 問題ID |
| `examType` | CLF / SAA / SAP |
| `questionText` | 問題文 |
| `choices[]` | 選択肢リスト |
| `correctAnswers[]` | 正解の選択肢 |
| `explanation` | 解説 |
| `tags[]` | 分野タグIDリスト |
| `isMultiple` | 複数選択かどうか |
| `createdAt` | 作成日時 |

GSI: `examType` をPKとして問題一覧を取得

---

#### ② Tags

| 属性 | 説明 |
|---|---|
| PK: `tagId` | タグID |
| `name` | 分野名 |
| `examType` | 対応する資格 |
| `officialRatio` | 公式出題比率（%） |

---

#### ③ QuestionTagRelations

| 属性 | 説明 |
|---|---|
| PK: `tagId` | タグID |
| SK: `questionId` | 問題ID |
| `examType` | 対応する資格 |

タグによる問題絞り込みを高速に行うための中間テーブル

---

#### ④ UserAnswers（生データ）

| 属性 | 説明 |
|---|---|
| PK: `userId` | ユーザーID |
| SK: `questionId#timestamp` | 問題ID＋回答日時 |
| `isCorrect` | 正解かどうか |
| `selectedAnswers[]` | 選択した回答 |
| `sessionId` | セッションID |
| `answeredAt` | 回答日時 |

---

#### ⑤ UserQuestionStats（問題ごとの集計）

| 属性 | 説明 |
|---|---|
| PK: `userId` | ユーザーID |
| SK: `questionId` | 問題ID |
| `correctCount` | 正解回数 |
| `incorrectCount` | 不正解回数 |
| `lastAnsweredAt` | 最終回答日時 |

不正解数フィルタ・苦手問題表示に使用

---

#### ⑥ UserTagStats（分野ごとの集計）

| 属性 | 説明 |
|---|---|
| PK: `userId` | ユーザーID |
| SK: `tagId` | タグID |
| `correctCount` | 正解回数 |
| `incorrectCount` | 不正解回数 |

分野別正答率の表示に使用

---

#### ⑦ Sessions

| 属性 | 説明 |
|---|---|
| PK: `userId` | ユーザーID |
| SK: `sessionId` | セッションID |
| `mode` | 演習 / 模試 |
| `examType` | 対応する資格 |
| `questionIds[]` | 出題問題IDリスト |
| `status` | active / completed / abandoned |
| `startedAt` | 開始日時 |
| `endedAt` | 終了日時 |
| `score` | 正答率 |
| `isPassed` | 合否 |
| `lastAnsweredAt` | 最終回答日時 |

---

#### ⑧ Reports

| 属性 | 説明 |
|---|---|
| PK: `questionId` | 問題ID |
| SK: `reportId` | 通報ID |
| `userId` | 通報者ID |
| `message` | 通報内容 |
| `reportedAt` | 通報日時 |

## 5. API設計

### 認証系

| メソッド | エンドポイント | 説明 |
|---|---|---|
| POST | `/auth/signup` | アカウント作成 |
| POST | `/auth/login` | ログイン |
| POST | `/auth/reset-password` | パスワードリセット |

---

### 問題系

| メソッド | エンドポイント | 説明 |
|---|---|---|
| GET | `/questions` | 問題一覧取得（フィルタ付き） |
| GET | `/questions/:id` | 問題1件取得 |
| GET | `/questions/:id/explanation` | 解説取得 |
| POST | `/questions/:id/report` | 問題を通報 |
| POST | `/questions/export` | 選択した問題のCSV生成・返却 |

**GET /questions　クエリパラメータ**

| パラメータ | 説明 |
|---|---|
| `examType` | CLF / SAA / SAP |
| `tagId` | 分野フィルタ |
| `minIncorrect` | 不正解数フィルタ（例：3以上） |
| `unanswered` | true/false（未回答のみ） |
| `limit` | 取得件数 |
| `shuffle` | true/false（ランダム順） |

---

### セッション系

| メソッド | エンドポイント | 説明 |
|---|---|---|
| POST | `/sessions` | セッション開始 |
| PUT | `/sessions/:id` | セッション終了・結果保存 |
| GET | `/sessions/:id` | セッション結果取得 |
| GET | `/sessions/active` | 未完了セッションの確認 |
| GET | `/sessions/:id/progress` | 途中までの回答状況取得 |

**途中再開のフロー**
```
ログイン
  └→ GET /sessions/active
        ├→ 未完了あり → 「再開しますか？」ダイアログ
        │      ├→ 再開 → GET /sessions/:id/progress → 途中から復元
        │      └→ 破棄 → PUT /sessions/:id（abandoned） → ホームへ
        └→ なし → ホームへ
```

---

### 回答系

| メソッド | エンドポイント | 説明 |
|---|---|---|
| POST | `/sessions/:id/answers` | 回答を記録 |

1問回答するごとに以下の3テーブルをトランザクション更新：
- `UserAnswers`（生データ追記）
- `UserQuestionStats`（正解/不正解カウントをインクリメント）
- `UserTagStats`（該当分野の正答率を更新）

---

### 統計系

| メソッド | エンドポイント | 説明 |
|---|---|---|
| GET | `/users/me/stats` | 分野別正答率・苦手分野一覧 |
| GET | `/users/me/question-stats` | 問題ごとの正解・不正解回数 |

---

## 6. 画面設計

### 画面一覧

#### 認証系

| # | 画面名 | 主な内容 |
|---|---|---|
| 1 | サインアップ | メール・パスワード入力、確認メール送信 |
| 2 | ログイン | メール・パスワード入力 |
| 3 | パスワードリセット | メールアドレス入力→再設定 |

#### メイン系

| # | 画面名 | 主な内容 |
|---|---|---|
| 4 | ホーム | 資格選択（CLF/SAA/SAP）、モード選択、学習状況サマリ |
| 5 | 演習設定 | 問題数指定、分野フィルタ、不正解数フィルタ |
| 6 | 模試設定 | 資格確認、問題数・時間の確認（変更不可） |
| 7 | 演習画面 | 問題・選択肢・即時正誤表示・解説・次の問題へ |
| 8 | 模試画面 | 問題・選択肢・タイマー・問題ナビゲーション |
| 9 | 結果画面 | スコア・合否判定・問題ごとの正誤一覧・解説展開 |

#### 学習管理系

| # | 画面名 | 主な内容 |
|---|---|---|
| 10 | マイページ | 分野別正答率グラフ、苦手分野ランキング |
| 11 | 問題一覧 | 未回答フラグ・チェックボックス・CSV出力・コピー・通報・解説展開 |

---

### 画面遷移

```
（未ログイン）
  └→ 問題一覧（閲覧・コピー・CSV出力のみ）

（ログイン済み）
  └→ ホーム
        ├→ 演習設定 → 演習画面 → 結果画面
        ├→ 模試設定 → 模試画面 → 結果画面
        ├→ マイページ
        └→ 問題一覧（未回答フラグ・通報も使用可）
```

## 7. 実装計画

### Phase 1　インフラ・認証基盤（約1〜2週）

1. Amplify + CodeCommit のセットアップ
2. Cognito ユーザープールの設定
3. DynamoDB テーブルの作成（8テーブル）
4. API Gateway + Lambda の雛形作成

### Phase 2　問題データ基盤（約1〜2週）

5. 問題・タグのDynamoDBスキーマ確定
6. サンプル問題データの投入
7. GET /questions API の実装
8. 問題一覧画面の実装（フィルタ・コピー・CSV出力）

### Phase 3　演習モード（約2週）

9. セッション開始・回答記録・終了APIの実装
10. 演習設定画面
11. 演習画面（1問ずつ・即時正誤表示）
12. 結果画面

### Phase 4　模試モード（約1〜2週）

13. 分野比率ロジックの実装（問題の比率配分）
14. 途中再開ロジックの実装
15. 模試設定画面
16. 模試画面（タイマー・ナビゲーション）
17. 結果画面（模試用：まとめて正誤＋解説）

### Phase 5　学習管理（約1週）

18. 統計APIの実装（分野別正答率・苦手問題）
19. マイページ画面（グラフ・苦手分野ランキング）

### Phase 6　通報・仕上げ（約1週）

20. 通報API + SES（メール送信）の実装
21. 通報ボタンUI
22. 全体のエラーハンドリング・バリデーション
23. 動作確認・デプロイ
